# 03 - User Personas

# Purpose

Define the primary audiences, their goals, motivations, pain points, and
how the platform should serve them. These personas guide UX, content,
and future feature decisions.

------------------------------------------------------------------------

# Persona 1 --- Aspiring Technical Writer

## Profile

-   Recent graduate or career switcher
-   Little or no industry experience
-   Learning documentation fundamentals
-   Preparing for first Technical Writing interview

## Goals

-   Build foundational knowledge
-   Understand industry terminology
-   Practice interview questions
-   Gain confidence

## Pain Points

-   Information scattered across blogs and videos
-   No structured practice
-   Doesn't know what skills employers expect

## MVP Features Used

-   Documentation Fundamentals
-   Style Guides
-   Daily Challenge
-   5-question Quick Practice
-   Answer explanations

------------------------------------------------------------------------

# Persona 2 --- Early Career Technical Writer

## Profile

-   1--3 years of experience
-   Works with documentation daily
-   Wants to grow into a Senior role

## Goals

-   Improve API documentation skills
-   Learn Docs-as-Code
-   Stay current with AI
-   Prepare for interviews

## Pain Points

-   Limited mentoring
-   Little time for learning
-   Difficulty identifying knowledge gaps

## MVP Features Used

-   Practice by Skill
-   10-question quizzes
-   AI for Technical Writers
-   API Documentation
-   Content Strategy

------------------------------------------------------------------------

# Persona 3 --- Experienced Technical Writer

## Profile

-   5+ years of experience
-   Works in SaaS or enterprise software
-   Interested in Staff or Lead positions

## Goals

-   Keep skills current
-   Refresh interview knowledge
-   Explore modern documentation trends

## Pain Points

-   Technology changes quickly
-   Existing learning resources are too basic
-   Few platforms focus on advanced documentation

## MVP Features Used

-   20-question mixed quizzes
-   Advanced topics
-   Professional Skills
-   Interview Preparation

------------------------------------------------------------------------

# Persona 4 --- Documentation Engineer

## Profile

-   Strong engineering background
-   Works with CI/CD, Git, APIs, automation
-   Blends software engineering and documentation

## Goals

-   Validate technical knowledge
-   Learn emerging AI workflows
-   Benchmark skills

## MVP Features Used

-   Docs-as-Code
-   Git & GitHub
-   REST APIs
-   Prompt Engineering
-   AI Automation

------------------------------------------------------------------------

# Common User Goals

-   Practice for 5--15 minutes per day
-   Learn something new every session
-   Receive immediate feedback
-   Understand why an answer is correct
-   Build confidence before interviews

------------------------------------------------------------------------

# User Motivations

-   Career growth
-   Interview readiness
-   Continuous learning
-   Professional credibility
-   Staying relevant in the AI era

------------------------------------------------------------------------

# UX Principles Derived from Personas

1.  Minimize friction.
2.  Allow users to start practicing within one minute.
3.  Keep quizzes short and focused.
4.  Explain every answer.
5.  Prioritize readability over visual decoration.
6.  Make the experience feel like a professional engineering tool, not a
    school exam.
7.  Encourage daily learning without overwhelming users.

# Design Implications

-   Homepage emphasizes "Sharpen your documentation skills every day."
-   Dashboard highlights Continue Practice, Daily Challenge, and
    Practice by Skill.
-   Skill-first navigation is more intuitive than tool-first navigation.
-   Professional, GitHub-inspired aesthetics reinforce credibility.
