# 09 - Brand Personality

# Purpose

Define the personality, tone, visual identity, and emotional experience
of the Technical Writing Practice Platform. This document is intended
for designers, developers, and AI design tools (Google Stitch, v0, Figma
AI).

------------------------------------------------------------------------

# Brand Positioning

**Sharpen your documentation skills. Every day.**

The platform is not an exam portal or an e-learning website. It is a
daily practice platform for modern Technical Writers.

It should feel like:

-   GitHub
-   Linear
-   Vercel
-   Raycast

It should **not** feel like:

-   Duolingo
-   Coursera
-   Udemy
-   Generic quiz websites

------------------------------------------------------------------------

# Brand Attributes

  Attribute      Description
  -------------- -------------------------------------------------
  Professional   Built for working professionals
  Technical      Comfortable for engineers and technical writers
  Minimal        Simple, clean, uncluttered
  Trustworthy    Accurate, consistent, reliable
  Modern         Reflects AI-era documentation practices
  Focused        Removes distractions and emphasizes learning

------------------------------------------------------------------------

# Voice & Tone

## Voice

-   Confident
-   Helpful
-   Precise
-   Encouraging
-   Never condescending

## UI Copy Examples

Good: - Start Practicing - Continue Practice - Practice by Skill - Daily
Challenge - Review Answers - Sharpen Your Skills

Avoid: - Great Job! - Awesome! - Keep Crushing It! - Fun Learning!

The tone should sound like a professional engineering product.

------------------------------------------------------------------------

# Emotional Goals

When a user opens the site they should think:

> "This is where I sharpen my documentation skills every day."

After completing a quiz:

> "I learned something useful."

------------------------------------------------------------------------

# Visual Direction

Design language:

-   GitHub-inspired
-   Documentation-first
-   White space over decoration
-   Monochrome palette
-   Subtle blue accent for primary actions
-   Rounded corners (6--8 px)
-   Thin borders
-   Minimal shadows

Avoid:

-   Glassmorphism
-   Neon colors
-   Large illustrations
-   Excessive animations
-   Gamified cartoon elements

------------------------------------------------------------------------

# Iconography

Use simple outline icons.

Preferred libraries:

-   Lucide
-   Heroicons
-   Tabler Icons

Icons should communicate function, not decoration.

------------------------------------------------------------------------

# Imagery

Minimal.

If illustrations are used, prefer:

-   Documentation
-   APIs
-   Flow diagrams
-   Abstract geometric patterns

Do not use mascot characters.

------------------------------------------------------------------------

# Messaging Framework

Headline:

Sharpen your documentation skills. Every day.

Supporting message:

Practice interview-ready questions, master modern documentation skills,
and stay current with AI, Docs-as-Code, and evolving technologies.

Primary CTA:

Start Practicing

------------------------------------------------------------------------

# Brand Principles

1.  Learning over testing.
2.  Skills over tools.
3.  Practice over memorization.
4.  Clarity over decoration.
5.  Consistency over novelty.
6.  Build confidence through repetition.

------------------------------------------------------------------------

# AI Design Guidance

When generating mockups:

-   Prioritize readability.
-   Keep layouts clean and spacious.
-   Use documentation-style typography.
-   Cards should feel lightweight.
-   Navigation should resemble a developer tool.
-   Components should be reusable and consistent.
-   Support both light and dark themes.
